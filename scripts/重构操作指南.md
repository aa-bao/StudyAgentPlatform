# 项目结构重构 - 手动操作指南

> 📅 创建时间：2025-01-17
> 🎯 目标：规范项目目录命名

---

## 📋 重构目标

| 原目录名 | 新目录名 |
|---------|---------|
| `KaoYanPlatform-back` | `java-back` |
| `KaoYanPlatform-front` | `vue-front` |
| `ai_service` | `python-back` |
| `ExtractQuestionFromPDF` | `python-back/tools/pdf-extractor` |

---

## ⚠️ 执行前准备

### 1. 关闭所有IDE和编辑器

**必须关闭：**
- VSCode / IntelliJ IDEA / PyCharm
- 所有终端窗口
- 任何打开项目文件的程序

**原因：** Windows文件锁定会导致重命名失败

### 2. 确认Git已备份

```bash
# 检查最后一次提交
git log -1

# 应该看到：backup: 重构前完整备份
```

---

## 🚀 执行方式（选择一种）

### 方式A：使用脚本（推荐）

#### Windows用户

```cmd
# 1. 进入项目根目录
cd F:\Coding\JavaProject\KaoYanPlatform

# 2. 运行重构脚本
scripts\refactor-structure.bat
```

#### Linux/Mac用户

```bash
# 1. 进入项目根目录
cd /path/to/KaoYanPlatform

# 2. 添加执行权限
chmod +x scripts/refactor-structure.sh

# 3. 运行重构脚本
./scripts/refactor-structure.sh
```

---

### 方式B：手动执行（如果脚本失败）

#### 步骤1：重命名Java后端

```cmd
# Windows CMD
move KaoYanPlatform-back java-back

# Windows PowerShell
Move-Item -Path "KaoYanPlatform-back" -Destination "java-back"

# Linux/Mac
mv KaoYanPlatform-back java-back
```

#### 步骤2：重命名Vue前端

```cmd
# Windows CMD
move KaoYanPlatform-front vue-front

# Windows PowerShell
Move-Item -Path "KaoYanPlatform-front" -Destination "vue-front"

# Linux/Mac
mv KaoYanPlatform-front vue-front
```

#### 步骤3：重命名Python后端

```cmd
# Windows CMD
move ai_service python-back

# Windows PowerShell
Move-Item -Path "ai_service" -Destination "python-back"

# Linux/Mac
mv ai_service python-back
```

#### 步骤4：移动PDF提取工具

```cmd
# Windows CMD
mkdir python-back\tools
move ExtractQuestionFromPDF python-back\tools\pdf-extractor

# Windows PowerShell
New-Item -ItemType Directory -Path "python-back\tools"
Move-Item -Path "ExtractQuestionFromPDF" -Destination "python-back\tools\pdf-extractor"

# Linux/Mac
mkdir -p python-back/tools
mv ExtractQuestionFromPDF python-back/tools/pdf-extractor
```

## 🐛 常见问题

### Q1: 重命名失败，提示权限拒绝

**A:** 关闭所有IDE和编辑器，确保没有程序占用文件

### Q2: Git没有识别到重命名

**A:** 执行 `git add -A` 让Git自动识别

### Q3: 某个文件丢失

**A:** 查看备份提交：`git log --all --full-history`

如果执行过程中遇到任何问题：
1. 停止操作
2. 记录错误信息

```bash
# 或查看备份
git log --oneline
```
