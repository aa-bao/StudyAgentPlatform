## 🤔 问题分析

### 1. 命名不规范

**问题：**
- ❌ `KaoYanPlatform-back` - 使用连字符
- ❌ `KaoYanPlatform-front` - 使用连字符
- ❌ `ExtractQuestionFromPDF` - 混合大小写，过长
- ❌ `ai_service` - 下划线风格

**影响：**
- 不同操作系统大小写敏感问题
- 不符合主流命名规范
- 影响专业度

### 2. 职责不清晰

**问题：**
- `ExtractQuestionFromPDF` - 离线工具
- `ai_service` - 在线服务
- 两者都是Python，功能重叠

**困惑：**
- 新人不知道哪个是生产服务
- 功能重复（都有GLM提取）

### 3. 数据混在项目根目录

**问题：**
- `data/` 直接在根目录
- 测试数据和项目代码混在一起

## 📋 详细命名方案

### 方案1：kebab-case（推荐）⭐⭐⭐

**风格：** 全小写 + 连字符

```
kaoyan-platform/
├── backend/
├── frontend/
├── ai-service/
└── tools/
    └── pdf-extractor/
```

**优点：**
- ✅ URL友好（http://localhost:8080 符合规范）
- ✅ 跨平台兼容（Windows/Mac/Linux）
- ✅ 符合Web项目主流
- ✅ GitHub项目常见风格

**参考：**
- https://github.com/facebook/react
- https://github.com/vuejs/core

---

### 方案2：snake_case

**风格：** 全小写 + 下划线

```
kaoyan_platform/
├── backend/
├── frontend/
├── ai_service/
└── tools/
    └── pdf_extractor/
```

**优点：**
- ✅ Python风格
- ✅ 数据库友好

**缺点：**
- ⚠️ URL不友好
- ⚠️ 不符合Web项目习惯

---

### 方案3：PascalCase

**风格：** 大驼峰

```
KaoYanPlatform/
├── Backend/
├── Frontend/
├── AiService/
└── Tools/
    └── PdfExtractor/
```

**优点：**
- ✅ Java风格

**缺点：**
- ❌ URL不友好
- ❌ 大小写敏感问题（Linux）
- ❌ 不符合微服务命名习惯

## 🔄 迁移步骤建议

### 阶段1：准备工作（不破坏现有结构）

```bash
# 1. 创建新目录结构（并行）
mkdir kaoyan-platform-new

# 2. 复制并重命名
cp -r KaoYanPlatform-back kaoyan-platform-new/backend
cp -r KaoYanPlatform-front kaoyan-platform-new/frontend
cp -r ai_service kaoyan-platform-new/ai-service
mkdir -p kaoyan-platform-new/tools/pdf-extractor
cp -r ExtractQuestionFromPDF/* kaoyan-platform-new/tools/pdf-extractor/

# 3. 更新配置文件
# - pom.xml (artifactId)
# - package.json (name)
# - Dockerfile
# - nginx配置
# - API调用路径
```

### 阶段2：验证测试

```bash
# 1. 测试后端
cd kaoyan-platform-new/backend
mvn clean install
mvn spring-boot:run

# 2. 测试前端
cd kaoyan-platform-new/frontend
npm install
npm run dev

# 3. 测试AI服务
cd kaoyan-platform-new/ai-service
pip install -r requirements.txt
uvicorn main:app --reload
```

### 阶段3：切换（一次性操作）

```bash
# 1. 备份当前项目
git add .
git commit -m "backup: 重构前备份"

# 2. 删除旧目录（可选）
# 或保留作为参考

# 3. 重命名新目录
mv kaoyan-platform kaoyan-platform-old
mv kaoyan-platform-new kaoyan-platform

# 4. 更新Git
git add -A
git commit -m "refactor: 重构项目目录结构"
```
